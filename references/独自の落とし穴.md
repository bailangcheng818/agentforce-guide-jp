> 方針：**技術の全詳細・全 CLI コマンド・Agent Script 構文は公式 `developing-agentforce` スキルに従う**（頻繁に更新されるので、ここでは重複保守しない）。
> このファイルが持つのは **(A) 公式に載っていない“独自の落とし穴”** と **(B) 組み立て時に毎回効く必須ルールの要点** だけ。番号は雛形（assets/patterns/*）のコメントと対応。

## (A) 独自の落とし穴（公式に無い・実プロジェクトで踏んだ）

| # | 落とし穴 | 自動でやること |
|---|---|---|
| 11 | 新規プロンプト道具が preview 実行時に **`MISSING_RECORD: No active template version`** | `<status>Published</status>` だけでは**有効版にならない**。deploy → retrieve で付与済み `<versionIdentifier>` を読み、トップレベルに **`<activeVersionIdentifier>` = その値** を追記して再 deploy。`versionIdentifier` は手で創作しない。**通常は①で人間がテンプレを用意・有効化済み**なので不要。②で受領したテンプレに有効版が無ければ**①へ差し戻し** |
| 12 | preview 実行時に **`REQUIRED_FIELD_MISSING: Missing required input parameter: Input:xxx`** | 入力は **`"Input:xxx"` 接頭辞付きで宣言**し `with "Input:xxx" = ...` で渡す。省くと planner は素の `xxx` を送りテンプレの必須入力名 `Input:xxx` と不一致。なおこの宣言は**テンプレが有効版を持って初めて** `preview start` 検証を通る（先に #11） |

### 自己診断：FunctionStep トレース（空回答・実行時失敗のとき）
```bash
TR=$(ls -t .sfdx/agents/<Api>/sessions/<session-id>/traces/*.json | head -1)
python3 -c "import json;t=json.load(open('$TR'));[print(s['function']) for s in t['plan'] if (s.get('stepType') or s.get('type'))=='FunctionStep']"
```
- `input` のキー名（`Input:xxx` か 素 `xxx` か → #12）／`errors[].detail` の `statusCode`（`MISSING_RECORD`→#11／`REQUIRED_FIELD_MISSING`→#12）。
- `errors` 無し・`input` 正常で空回答 → **検索が0件**の可能性 → ①（テンプレ/検索）へ差し戻し。

## (B) 組み立て時の必須ルール（要点・詳細は developing-agentforce）

| # | 要点 |
|---|---|
| 1 | プロンプト道具：出力に `is_used_by_planner: True`（+`is_displayable: True`）、reasoning で `set @variables.x = @outputs.promptResponse` |
| 2 | **社内向け**＝`default_agent_user`／MessagingSession 連結変数／escalation／`connection messaging:` を**外し**、`agent_type: "AgentforceEmployeeAgent"`。お客様向けは Service＋実行ユーザー |
| 3 | 構文：4スペース・`True`/`False`・ダブルクオート・ハブ&スポーク（別ルーター作らない） |
| 4 | `default_locale: "en_US"`（検証済み）。表示は instructions/messages の日本語で担保 |
| 5 | 入力 `"Input:xxx"`（テンプレ apiName と一致）／出力 `promptResponse` 固定 |
| 13 | `target:` を持つアクションには **`outputs:` ブロック必須**（無いと publish で Internal Error） |
| 16 | プランナーは**全アクションを起動時検証**。サービス時はカスタム権限セットに呼ぶ Apex 全クラス＋オブジェクト権限を漏れなく |
| 20 | welcome は**1行・静的**（改行/変数は最初の担当の instructions 側で） |
