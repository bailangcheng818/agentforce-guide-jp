# 環境セットアップ（初回のみ・コードが分からない人向け）

対話を始める前に、実行エージェント（あなた）が**自動でチェック**し、足りないものがあれば**やさしい日本語で1つずつ案内**する。ユーザーに技術用語や生のエラーは見せない。

> 方針：CLI とログインは「**一度だけ**」の準備。済んでいれば毎回スキップ。
> ユーザーが自分でできない場合は「社内のIT担当に頼む」案内に切り替えてよい。

---

## チェック1：Salesforce CLI が入っているか

```bash
sf --version || npx -y @salesforce/cli --version
```
- ✅ バージョンが出れば OK（次へ）。
- ❌ 出なければ → ユーザーにこう案内する（一度に一つ）：
  > 「最初に、Salesforce とやり取りする小さな道具をインストールします。下のコマンドをターミナルに貼って実行してください。」
  - Mac / Windows 共通（Node.js が入っている場合）：
    ```bash
    npm install -g @salesforce/cli
    ```
  - Node.js が無い場合：先に https://nodejs.org からインストーラを入れてもらう（Windows は https://developer.salesforce.com/tools/salesforcecli の `.exe` でも可）。
  - 入れた後：「`sf --version` でバージョンが出たら教えてください」と確認。

## チェック2：組織（org）にログインしているか

```bash
sf org list --json
```
- ✅ 目的の組織が `Connected` で出れば OK。
- ❌ 出ない／期限切れなら → こう案内する：
  > 「次に、あなたの Salesforce にログインします。下のコマンドを実行するとブラウザが開くので、いつもの ID とパスワードで入ってください。」
  ```bash
  sf org login web --alias agentforce-demo
  ```
  - ⚠️ 組織のドメインが分かっている場合：`--instance-url https://<ドメイン>.my.salesforce.com` を付ける。
  - 分からなければ案件オーナー／IT 担当に確認してもらう。

## チェック3：既定の組織を固定

```bash
sf config get target-org --json
```
- 未設定なら：
  ```bash
  sf config set target-org agentforce-demo
  ```
- これで以降のコマンドが自動でこの組織を使う。

---

## ユーザーへの言い方（トーン）

- 「準備はこの3つだけです。一度やれば次からは要りません。」と安心させる。
- 1コマンドずつ。実行結果を聞いてから次へ。
- エラーが出ても生ログは見せず、「ここを直しますね」と引き取る。
- どうしても進まない（権限・社内ポリシー）→「社内のIT担当にこの3つの準備をお願いしてください」と案内。
