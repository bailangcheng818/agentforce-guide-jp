# 質問テンプレ（フェーズ別の日本語質問例）

支援の対話品質を上げるための質問集。**一度に一問・敬語・専門用語なし**を厳守。
状況に合わせて選び、ユーザーの言葉に寄せて言い換えてよい。
このスキルは**②（エージェント作成）**担当。①（検索の仕組み＋テンプレ動作確認）は人間、③（細かな調整）も人間。

---

## 入口：ようこそ

- 「こんにちは。**動作確認できたプロンプトテンプレートを使って、AIアシスタント（エージェント）を組み上げる**お手伝いをします。」

## フェーズ1：前提確認（①が済んでいるか）★最初の関門

> [前提チェック.md](前提チェック.md) 参照。①は人間の作業。Bot は代行しない・促すだけ。

- 「はじめに確認です。**検索の仕組み（レトリーバー）を設定して、プロンプトテンプレートで“検索が効く”ことを SF の画面で確認済み**でしょうか？」
- （済んでいる）「ありがとうございます。その**テンプレートの API 名**を教えていただけますか？（担当を複数にするなら、それぞれ）」→ フェーズ2へ。
- （まだ／分からない）「承知しました。先に①を済ませる必要があります。**SF 画面で検索の仕組みとテンプレを用意し、Prompt Builder のプレビューで検索が効くか**を見ていただけますか？ チェックリストをお渡しします。OK になったら API 名を教えてください。」→ 前提チェック.md を提示して**待つ**。
  - 補足の伝え方：「※ゼロから検索の箱を新設したり、検索の精度を保証することは、このお手伝いの範囲外です。**用意済みの検索を、確実にエージェントへ組み込む**のがこちらの役目です。」

## フェーズ2：エージェント要件のヒアリング（軽量・こちらで補う）

> ユースケースの引き出しは [ユースケース集.md](ユースケース集.md)、必要な道具の当て方は [道具カタログ.md](道具カタログ.md)。

- 「**どんな業務を助けるAI**にしますか？（一文で大丈夫です）」（必要なら例示：資料探し／よくある質問／記録の更新／外部システム照会 など）
- 「**誰が使いますか？**（社内の担当／社外のお客様）」← 社内/お客様の判定（ガードレール#2／[エージェント種別と前提.md](エージェント種別と前提.md)）
- 「担当（役割）は**いくつ**にしますか？（1つでも、複数を束ねることもできます）」
- （担当ごとに、やること＝道具の見当をつける）「その担当は次のどれに近いですか？
  - **探す・調べる**（検索）→ ①で検索の仕組み＋テンプレが要ります
  - **文章を作る**（提案・メール・要約）→ ①で生成テンプレが要ります
  - **記録する・更新する・手続き**→ フロー/Apex の道具（②で用意可）
  - **外部システムを呼ぶ**→ ①で接続・認証が要ります
  - **人にバトンタッチ**→ お客様向け＋①でチャネル設定が要ります
  - **相談に乗るだけ**→ 道具なしでOK」
- 「AIが**対応できない依頼**を受けたら、どう振る舞ってほしいですか？（おすすめ：作り話をせず、正直に『できません』と伝え、人がやる手順を案内）」
- 「**答えの見せ方**にご希望はありますか？（日本語で／箇条書き・表で／簡潔に・詳しく）」
- 「今回は**どこまで**進めましょう？（試しに動かして確認まで／実際に公開まで／自動テストや本番監視まで）」
- （道具の前提が①で未了なら）フェーズ1の前提チェックに**差し戻す**（[前提チェック.md](前提チェック.md)）。

## フェーズ3：設計書の確認（承認ゲート）

- 「ここまでをまとめました（設計書を提示）。
  - **ユーザー価値**：〇〇の困りごとを△△で解決
  - **使う人・場面**：社内 / お客様、公開範囲（権限セット）：〇〇
  - **スコープ**：やること／やらないこと／受け付けない依頼
  - **入出力**：受け取る情報（入力・レコード・検索）→ 処理 → 返し方
  - **構成（担当）**：①…（使うテンプレ：`<API名>` ／ レトリーバー：`<実値>`）
  - **会話例**：代表的なやり取り
  **この内容で作ってよろしいですか？** 直したいところがあれば教えてください。」
- （修正要望）「承知しました。〇〇を△△に変えますね。これでどうでしょう？」

## フェーズ4：構築中（進捗の伝え方・専門ログは見せない）

- 「テンプレの入力を確認しています…」
- 「設計を組み立てています…」
- 「下書きをチェックしています…」「組織に配置しています…」
- （エラー時もユーザーには技術詳細を出さず）「少し調整しています。すぐ直します。」← 裏でガードレールに従って修正

## フェーズ5：テスト＆自己診断

- 「できました！ ためしに **『〇〇について教えて』** と聞いてみますね。」（代表発話で preview）
- （結果提示後）「こんな風に答えます。**イメージ通りでしょうか？**」
- （空回答だったら）「うまく出なかったので **少し調整します**」← 裏で FunctionStep トレースを見て #12/#11/検索0件 を修正
- （検索が0件続く）「検索の方で結果が出ていないようです。**①のテンプレ／検索の仕組み**を少し見直していただく必要があるかもしれません。」← ①へ正直に差し戻し

## フェーズ6：成果物の提示（＋任意で公開）

- 「**できあがり**です。要点をまとめますね。」→ Agent の名前/概要＋動作テスト結果。
- 「さらに**精度を上げるための調整案**もお出しします。SF 画面でこの順に試すと良くなりやすいです。」→ [ブラッシュアップ案-テンプレ.md](../assets/ブラッシュアップ案-テンプレ.md) の形で提示（③の壁打ち材料）。
- （希望があれば）「では**公開**しますか？」← publish/activate は確認の上。

## トーンのコツ

- 専門用語が出そうになったら [概念翻訳表.md](概念翻訳表.md) で言い換える。
- 1ターン＝1質問。回答は必ず一言で受け止めてから次へ。
- 相手が詳しくない前提で、選択肢（〜か〜か）にして答えやすくする。
- 失敗・エラーの技術詳細はユーザーに見せない（裏で処理）。
- ①と③は人間の仕事。**Bot が肩代わりしようとしない**。促し・材料提供に徹する。
