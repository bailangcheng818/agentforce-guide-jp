# 概念翻訳表（Agentforce 用語 ↔ 業務日本語）

向導の核。**ユーザーには右側（やさしい日本語）だけを見せ**、内部で左側（Agentforce 用語）にマップする。

| Agentforce 用語 | ユーザーに見せる日本語 | ひとことの説明（必要時のみ） |
|---|---|---|
| Agent | **AIアシスタント** | 全体。「〇〇を助けるAI」 |
| Subagent / Topic | **担当（たんとう）** | AIの中の役割分担。「資料探し担当」など |
| agent_router | **受付（うけつけ）** | 入力を見て担当に振り分ける係（自動で用意） |
| Action / Tool | **道具（どうぐ）／できること** | 担当が使える1つの動作 |
| Prompt Template action | **AIに文章を書かせる道具** | 検索結果・提案文・要約などを生成 |
| Flow action | **自動処理の道具** | レコードの取得・更新など決まった処理 |
| Apex action | **専用プログラムの道具** | 業務ロジックを呼ぶ道具 |
| External Service / API action | **外部システムを呼ぶ道具** | 外部APIを呼ぶ（要：認証の用意） |
| Standard action | **既製の道具** | メール送信・タスク作成など標準機能 |
| Knowledge / Retriever / RAG | **資料の引き出し（検索）** | 資料・過去データを検索する元 |
| Data Library / Grounding | **根拠データ／根拠付け** | 答えのよりどころにする資料 |
| Guardrails | **守りごと（やってよい/だめ）** | AIの振る舞いの境界 |
| Instructions | **指示（しじ）** | 担当・道具への日本語の指図 |
| Einstein Trust Layer | **安全フィルター** | マスキング・監査の仕組み（自動） |
| Escalation | **人にバトンタッチ** | 有人対応へ引き継ぐ（お客様向けのみ） |
| Agent Spec | **設計書** | 承認用にAIがまとめる日本語の設計書（ユーザー価値〜会話例） |
| `validate` / compile | **下書きチェック** | 文法が通るか確認 |
| `deploy` | **組織に配置** | Salesforce に置く |
| `preview` | **試し会話** | 実際に話して動作確認 |
| `publish` / `activate` | **公開・有効化** | お客様/社内に使えるようにする |
| Employee agent | **社内向け** | 社員が使う。default_agent_user なし |
| Service agent | **お客様向け** | 顧客対応チャネル。Einstein Agent User 必要 |
| `off_topic` / `ambiguous_question` | **範囲外のとき／あいまいなとき の担当** | 自動で用意する標準の担当 |

## 言い換えの方針

- 「サブエージェント」「アクション」「フロー」「プランナー」等は**口に出さない**。
- どうしても概念を説明する必要があるときだけ、右端の「ひとことの説明」を使う。
- ユーザーが自分から英語/SF用語を使った場合は合わせてよいが、こちらからは持ち出さない。
