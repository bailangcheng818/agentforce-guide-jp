# 役割分担マトリクス（人間 ↔ AI/Bob の線引き）

このスキルの背骨。Agentforce の構築は**3工程**に分かれ、工程ごとに担当が違う。
**①人間（SF画面・組織設定）→ ②Bob（CLI・Agent Script）→ ③人間（SF画面で微調整）**。

> 基本原則：**「組織の土台・外部連携・認証・データ基盤・チャネル」は人間（①）**、**「エージェント本体の組み立て・道具のバインド・検証・デプロイ・テスト・改善案」は Bob（②）**、**「精度の作り込み・安全性の最終判断」は人間（③）**。
> 境界は Salesforce のリリースで**メタデータ/CLI 側に動く**傾向。固有の前提は対象組織で都度確認する。

## 工程の全体像

| 工程 | 担当 | 中身 |
|---|---|---|
| **① 土台づくり** | **人間（SF画面）** | データ基盤・検索・テンプレ・認証・チャネル・実行ユーザー等の用意と**動作確認** |
| **② エージェント構築** | **Bob（CLI）** | `.agent` 作成→道具バインド→validate→deploy→preview→（公開）→ブラッシュアップ案 |
| **③ チューニング** | **人間（Bob/LLM 壁打ち＋SF画面）** | 指示・検索・出力の作り込み、安全性の最終確認、本番監視 |

## マトリクス（capability 別）

| capability | ① 人間（SF画面で用意・確認） | ② Bob（CLI/Agent Script で自動化） | ③ 人間（仕上げ） |
|---|---|---|---|
| **エージェント本体（.agent）** | — | **生成・記述・validate・deploy・preview・publish・activate** | 指示文の作り込み |
| **担当（トピック/サブ）構成・振り分け** | — | **ハブ&スポーク等で自動構成** | 振り分け精度の調整 |
| **プロンプト道具**（`generatePromptResponse://`） | テンプレ作成＋**Prompt Builder で動作確認**＋有効化 | テンプレを**バインド**（入力名 `Input:xxx` 一致） | 本文・出力形式の調整 |
| **レトリーバー/RAG**（検索・グラウンディング） | **Data Cloud・検索インデックス・レトリーバー作成＋確認**（CLIで新設・一覧不可） | レトリーバーを**参照**（`retriever://` or テンプレ埋め込み） | 関連度・件数の調整 |
| **ナレッジ/Data Library** | **ナレッジ取り込み・Data Library 作成・紐付け** | グラウンディング前提のテンプレ/道具を組む | 取り込み内容の見直し |
| **フロー道具**（`flow://`） | （任意）Flow を画面で作る場合 | **`generating-flow` で Flow 生成→deploy→`flow://` でバインド** | 例外処理の調整 |
| **Apex 道具**（`apex://`） | — | **`generating-apex` で `@InvocableMethod` 生成→deploy→`apex://` でバインド** | ロジックの調整 |
| **外部API道具**（`externalService://`） | **Named Credential の認証（OAuth/シークレット）・接続承認** | External Service 登録（OpenAPI）→`externalService://` でバインド | 認証・エンドポイント確認 |
| **標準アクション**（`standardInvocableAction://` 等） | （多くは既存機能） | **標準アクションをバインド** | — |
| **構造化入出力（CLT）** | — | `generating-custom-lightning-type` でスキーマ生成→deploy | 複雑UIは人間がLWC |
| **人手へのエスカレーション** | **Omni-Channel／メッセージングチャネルの設定**（画面のみ） | サービスエージェントに `@utils.escalate`＋`connection messaging:` を記述 | ルーティング確認 |
| **エージェント種別の前提** | **Einstein Agent User 作成＋ライセンス付与**（サービス時、画面要素あり）、権限セット付与判断 | 権限セット XML 生成・deploy・assign、config の `agent_type` 設定 | 権限の過不足確認 |
| **テスト** | テスト観点・期待値の決定、安全性の判定 | **テスト spec(YAML) 生成→`sf agent test`→結果解析** | 失敗の解釈・修正判断 |
| **本番監視（観測）** | **STDM（Agentforce Activity）有効化**（Data Cloud） | `observing-agentforce` でトレース取得・問題分類 | 改善方針の決定・再公開承認 |
| **公開（publish/activate）** | 公開可否の最終承認 | コマンド実行（確認の上） | 公開後の確認 |

## 「② Bob ができる」ことの要約

- `.agent` の生成・記述・**validate / deploy / preview（--use-live-actions）/ publish / activate**。
- 既存の道具（テンプレ・フロー・Apex・外部サービス・レトリーバー・標準アクション）の**バインド**。
- 不足する Apex/Flow/CLT/権限セットの**生成とデプロイ**（別スキルへ委譲）。
- テスト spec の生成・実行・解析、トレースでの**自己診断**、**ブラッシュアップ案**の作成。

## 「① 人間がやる（Bob は促すだけ）」ことの要約

- **データ基盤と検索**：Data Cloud / 検索インデックス / レトリーバー / Data Library の作成・有効化（CLIで新設・一覧できない）。
- **生成テンプレの作成と動作確認**（Prompt Builder で検索が効くか・出力品質を目視）。
- **外部連携の認証**：Named Credential / 接続 / OAuth / シークレット。
- **チャネル展開**：メッセージング・Web/埋め込み・音声・メール・Slack 等。
- **実行ユーザーとライセンス**：Einstein Agent User、Einstein Agent ライセンス付与。
- これらが未了だと②でつまずく → **②に入る前に [前提チェック.md](前提チェック.md) で確認**。

## 「③ 人間がやる」ことの要約

- 指示文・検索クエリ・出力形式の**作り込み**（Bob/LLM と壁打ち、SF画面で操作）。
- **安全性の最終判断**（[ガードレール.md](ガードレール.md) のセーフティ観点）。
- **本番監視**の結果に基づく改善・再公開の承認。
