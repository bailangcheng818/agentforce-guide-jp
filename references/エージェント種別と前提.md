# エージェント種別と前提（社内向け / お客様向け）

エージェントには種別があり、種別ごとに**必要な前提（①人間）**と**`.agent` の書き方（②Bob）**が変わる。
フェーズ2で「誰が使うか」を聞いて判定する（ガードレール#2）。

## 種別の判定

| 種別（config の `agent_type`） | 誰が使う | 実行ユーザー | 代表ユースケース |
|---|---|---|---|
| **従業員エージェント** `AgentforceEmployeeAgent` | 社内の社員 | ログイン中の社員（その人の権限で動く＝共有ルールを尊重） | 社内Q&A、業務支援、レコード更新、IT/HR |
| **サービスエージェント** `AgentforceServiceAgent` | 社外のお客様 | 専用の Einstein Agent User（固定権限） | 問い合わせ対応、ケース対応、注文照会 |

> ほかに、Salesforce が出す**用途別テンプレ**（SDR / セールスコーチ / パーソナルショッパー 等）があり、標準トピック・アクションが付属する。本スキルは Agent Script で**ゼロから組む**経路。テンプレ前提のユースケースは [ユースケース集.md](ユースケース集.md) を当てに、必要な道具を②で組む。

## 従業員エージェント（社内向け）— 既定でおすすめ

**`.agent`（②Bob）**
- `config: agent_type: "AgentforceEmployeeAgent"`。
- **付けない**もの（付けると publish が「Internal Error」/ 黙って失敗）：
  - `default_agent_user`
  - MessagingSession 連結変数（`EndUserId` / `RoutableId` / `ContactId` / `EndUserLanguage`）
  - `connection messaging:` ブロック（＝`@utils.escalate` も使わない。人手対応は「ケース作成の道具」や担当の切替で代替）

**前提（①人間）**
- 利用する社員に**権限セット**を付与（その人が呼ぶ Apex/オブジェクトへのアクセス）。
- Einstein Agent ライセンス（組織で有効化済み前提）。

## サービスエージェント（お客様向け）

**`.agent`（②Bob）**
- `config: agent_type: "AgentforceServiceAgent"` ＋ **`default_agent_user: "<実行ユーザー>"`（必須）**。
- MessagingSession 連結変数を定義。人手へは `@utils.escalate`＋`connection messaging:`。

**前提（①人間／一部②でCLI可）**
- **Einstein Agent User の作成**（組織固有の命名）と **Einstein Agent ライセンス付与**（画面要素あり）。
- システム権限セット `AgentforceServiceAgentUser` の付与（CLIで `sf org assign permset` 可）。
- **カスタム権限セット**：エージェントが呼ぶ**すべての Apex クラス**を `<classAccesses>` に列挙し、必要な**オブジェクト権限**も付与（XML は②で生成、付与は CLI 可）。
- **メッセージング/Omni-Channel チャネル**の設定（画面のみ）。

> 重要（落とし穴）：プランナーは起動時に**全担当の全アクション**を検証する。1つでも権限不足だとエージェント全体が失敗する。`WITH USER_MODE` のApexで該当オブジェクト権限が無いと**0件で黙って失敗**する。→ 権限セットは漏れなく。

## どちらでも共通の前提（①人間）
- 道具が依存する基盤：検索（レトリーバー/Data Cloud）、生成テンプレ、外部連携の認証、ナレッジ/Data Library 等は**①で用意・確認**（[前提チェック.md](前提チェック.md)）。

## サービスエージェントの実行ユーザー確認・作成（②でCLI補助）
```bash
# 既存の Einstein Agent User を探す
sf data query --json -q "SELECT Username, IsActive FROM User WHERE Profile.UserLicense.Name = 'Einstein Agent' AND IsActive = true LIMIT 5"
# プロファイル ID
sf data query --json -q "SELECT Id FROM Profile WHERE Name = 'Einstein Agent User'"
# システム権限セット付与
sf org assign permset --json --name AgentforceServiceAgentUser --on-behalf-of <user>
# カスタム権限セット deploy & 付与
sf project deploy start --json --metadata PermissionSet:<PS名>
sf org assign permset --json --name <PS名> --on-behalf-of <user>
```
- ライセンス付与・チャネル設定は**画面**。実行ユーザーが未整備だと publish や preview で失敗する（[ガードレール.md](ガードレール.md) #9 含む）。
