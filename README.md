# agentforce-guide-ja

Agentforce エージェントを **日本語の業務会話だけ** で組み立てるための Agent Skill（Coding Agent 用）。
Agentforce を専門としないコンサルタントが、専門用語に触れずにエージェントを設計・構築できるよう支援する。
技術の詳細は公式 `developing-agentforce` スキルへ委譲し、本スキルは **業務視点の進め方・言い換え・独自の落とし穴** だけを持つ。

## 適用範囲

- **対象ユーザー**：Agentforce 非専門のコンサルタント・業務担当
- **対象タスク**：営業 / サービス / マーケ / コマース / 社内（IT・HR）向け Agentforce エージェントの要件整理・設計・組み立て・デプロイ・テスト
- **2つの入口**
  - **入口A**：すでにある設計書 / 要件メモを渡して組み立てる
  - **入口B**：ゼロから対話で要件を発散・収束し、設計書を生成してから組み立てる
- **人と AI の分担**：土台づくり（検索・レトリーバー・テンプレの動作確認）と最終調整は **人**、組み立て（構築・デプロイ・テスト）は **AI が自動化**

## 適用外（やらないこと）

- レトリーバー / Data Cloud / 外部連携の **新設** や検索精度の保証（＝人が SF 画面で用意する領域）
- 画像 / 動画 / 音声の生成、アルタイム制御 など Agentforce の範囲外のもの

## 構成

| パス                          | 内容                                                         |
| ----------------------------- | ------------------------------------------------------------ |
| `SKILL.md`                  | エントリ。人と AI の区分け・全体フロー・最重要ルール         |
| `references/`               | 進め方・質問例・用語の言い換え・前提チェック・独自の落とし穴 |
| `assets/設計書-テンプレ.md` | 2 入口が合流する設計書テンプレート                           |
| `assets/patterns/`          | `.agent` 雛形・プロンプトテンプレ雛形                      |

## 前提

- Salesforce CLI（`sf`）
- Agentforce が利用できる Salesforce 組織
- 公式 `developing-agentforce` スキル（技術詳細の委譲先）

## 使い方

Coding Agent のスキルとして配置し、「営業を助ける AI を作りたい」「設計書から組み立てたい」などと話しかけると起動する。
